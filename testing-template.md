# Bug Report: Mixed prices returned when sorting low to high

## Summary
On the men's shirts category page, sorting products by price from low to high returns mixed price order.

## Environment
- Device: Desktop
- Browser: Chrome
- OS: Windows 11

## Preconditions
- User is on the men's shirts category page
- Products with different prices are visible

## Steps to Reproduce
1. Open the men's shirts category page
2. Select sorting option "Low to High"
3. Observe the displayed products

## Expected Result
Products should appear in ascending order by price.

## Actual Result
Products appear in mixed order and are not fully sorted from lowest to highest.

## Severity
High

## Priority
High

## Notes
This affects product discovery and creates confusion for users.
